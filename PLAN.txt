Handling one `.shader` with multiple include files

What Unity data gives us
- Compiled bytecode + metadata, not original source include boundaries.

What we can reconstruct
1) A behavior-equivalent single-file shader (primary output).
2) Optional reconstructed include mode:
   - extract repeated helper blocks into generated `.cginc`/`.hlsl`
   - emit generated `#include` lines
   - add provenance comments for reconstructed chunks.

What cannot be guaranteed
- Original include filenames, split points, comments, and formatting.

Practical target
- Behavior fidelity first, then clean modular source output.
